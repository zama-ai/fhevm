# Live testnet runbook: Sepolia + Polygon Amoy

End-to-end procedure for spinning up the full FHEVM host stack on two EVM
testnets, wiring the `ConfidentialBridge` between them with production-grade
LayerZero V2 settings, running the mock coprocessor daemon, and exercising the
deployment with `FHECounter`.

Target chains used as the running example:

| Chain            | EVM chainId | LZ V2 EID |
| ---------------- | ----------- | --------- |
| Ethereum Sepolia | `11155111`  | `40161`   |
| Polygon Amoy     | `80002`     | `40267`   |

The procedure generalises to any chain pair that has a canonical LayerZero V2
endpoint. The LZ V2 endpoint address is the same on every chain:
`0x6EDCE65403992e310A62460808c4b910D972f10f`.

> **Scope.** This runbook covers the host-contract side of the stack plus the
> mock coprocessor needed to make FHE operations observable in plaintext. The
> production coprocessor (signers, KMS endpoints, …) is out of scope — see
> the protocol RFCs for that.

---

## Prerequisites

### Wallets and funds

A single deployer key is reused across both chains. Faucets:

- Sepolia: <https://www.alchemy.com/faucets/ethereum-sepolia>
- Polygon Amoy: <https://faucet.polygon.technology/>

### `.env` (project root)

```bash
DEPLOYER_PRIVATE_KEY=0x...                                       # funded on BOTH chains
LZ_ENDPOINT_ADDRESS=0x6EDCE65403992e310A62460808c4b910D972f10f   # canonical LZ V2 endpoint
SEPOLIA_RPC_URL=https://sepolia.drpc.org                          # also read by the daemon
POLYGON_AMOY_RPC_URL=https://rpc-amoy.polygon.technology          # also read by the daemon
ETHERSCAN_API_KEY=...                                            # optional, block-explorer verification
# Plus every variable the host-stack deploy already requires (KMS nodes,
# coprocessor signers, pausers, thresholds, …) — see .env.example.
```

`RPC_URL` is set per-invocation rather than once in `.env` (`hardhat.config.ts`
reads it at call time, see `getChainConfig`). The `SEPOLIA_RPC_URL` /
`POLYGON_AMOY_RPC_URL` entries are read by the mock coprocessor daemon.

### One-time installs

```bash
pnpm install                          # parent project (ethers v6)
cd lz-wiring && pnpm install --ignore-workspace && cd ..   # LZ CLI workspace (ethers v5)
```

`--ignore-workspace` is required so `pnpm` doesn't hoist `lz-wiring`'s ethers v5
into the parent's `node_modules` and break the host-stack hardhat config.

## Step 1 — Deploy host stack on Sepolia

```bash
RPC_URL="$SEPOLIA_RPC_URL" \
  npx hardhat --network sepolia task:deployAllHostContracts --with-kms-generation true

mkdir -p addresses-sepolia
cp addresses/{FHEVMHostAddresses.sol,.env.host} addresses-sepolia/
```

The `cp` is **critical** — the next chain's deploy overwrites `addresses/`.

When `LZ_ENDPOINT_ADDRESS` is present in the env, `task:deployAllHostContracts`
also upgrades the bridge empty proxy to its real implementation at the end of
the run, so at this point Sepolia has a fully deployed host stack including
`ConfidentialBridge`.

## Step 2 — Deploy host stack on Polygon Amoy


Note each chain's `CONFIDENTIAL_BRIDGE_CONTRACT_ADDRESS` from

## Step 3 — Wire the bridges (production: LayerZero CLI + dstChainId)

`lz:oapp:wire` (in `lz-wiring/`) handles peers, send/receive libraries, DVNs,
Executor config, confirmations and enforced options. It does **not** set the
bridge-specific `dstChainId` mapping (an FHEVM extension to the OApp
interface), so there's an extra call per chain at the end.

> Why `lz-wiring/` rather than calling `lz:oapp:wire` from the root project?
> `@layerzerolabs/toolbox-hardhat` transitively depends on `zksync-web3`,
> which is hard-coded against ethers v5. The parent project is on ethers v6.
> `lz-wiring/` is an isolated `pnpm` workspace that pins ethers v5 and
> operates on the already-deployed bridges via `hardhat-deploy` artifacts
> generated by `lz-wiring/scripts/sync-deployments.ts`.

### 3.1 — Export the bridge addresses

```bash
export SEPOLIA_BRIDGE_ADDRESS=$(grep '^CONFIDENTIAL_BRIDGE_CONTRACT_ADDRESS=' addresses-sepolia/.env.host | cut -d= -f2)
export POLYGON_AMOY_BRIDGE_ADDRESS=$(grep '^CONFIDENTIAL_BRIDGE_CONTRACT_ADDRESS=' addresses-amoy/.env.host | cut -d= -f2)
echo "Sepolia bridge: $SEPOLIA_BRIDGE_ADDRESS"
echo "Amoy bridge:    $POLYGON_AMOY_BRIDGE_ADDRESS"
```

`lz-wiring/` auto-loads the parent `.env`, so `DEPLOYER_PRIVATE_KEY`,
`SEPOLIA_RPC_URL`, and `POLYGON_AMOY_RPC_URL` are already picked up.

### 3.2 — Dry-run the wiring

```bash
cd lz-wiring
pnpm wire:dry
```

Prints the diff between current on-chain state and the desired state encoded
in `layerzero.config.ts`, for **both** chains in a single run:

- `setPeer(remoteEid, addressToBytes32(remoteBridge))` per direction
- `setSendLibrary` / `setReceiveLibrary` to the canonical LZ V2 libraries
- `setConfig` for the Executor + ULN (DVNs + confirmations)
- `setEnforcedOptions` for `lzReceive` gas

Review the diff. If anything's unexpected — wrong DVN, wrong gas, wrong remote
address — edit `lz-wiring/layerzero.config.ts` and re-run `pnpm wire:dry`
before continuing.

### 3.3 — Apply the wiring

```bash
# still inside lz-wiring/
pnpm wire
```

The CLI submits one tx per change, on each chain, signed by
`DEPLOYER_PRIVATE_KEY`. It pauses after each batch — press `y` to continue.
After it finishes both bridges are LayerZero-wired and can send/receive
messages.

### 3.4 — Set the bridge-specific `dstChainId`

```bash
cd ..   # back to host-contracts root

# Sepolia → Amoy
cp addresses-sepolia/{FHEVMHostAddresses.sol,.env.host} addresses/
RPC_URL="$SEPOLIA_RPC_URL" \
  npx hardhat --network sepolia task:wireBridge \
  --remote-eid 40267 \
  --remote-bridge "$POLYGON_AMOY_BRIDGE_ADDRESS" \
  --remote-chain-id 80002

# Amoy → Sepolia
cp addresses-amoy/{FHEVMHostAddresses.sol,.env.host} addresses/
RPC_URL="$POLYGON_AMOY_RPC_URL" \
  npx hardhat --network polygonAmoy task:wireBridge \
  --remote-eid 40161 \
  --remote-bridge "$SEPOLIA_BRIDGE_ADDRESS" \
  --remote-chain-id 11155111
```

> `task:wireBridge` re-sends `setPeer` here — idempotent but a wasted gas tx
> because `pnpm wire` already set it. If gas matters, skip `task:wireBridge`
> and call `setDstChainId` directly from a `hardhat console`:
>
> ```bash
> RPC_URL="$SEPOLIA_RPC_URL" npx hardhat --network sepolia console
> ```
>
> ```javascript
> const b = await ethers.getContractAt("ConfidentialBridge", process.env.SEPOLIA_BRIDGE_ADDRESS);
> await (await b.setDstChainId(40267, 80002)).wait();
> ```

### 3.5 — Verify both sides

```bash
cd lz-wiring
pnpm peers:get      # peers on both chains
pnpm config:get     # libraries + DVN + Executor on both chains
cd ..
```

`peers:get` should show, for each chain, the remote bridge address
(left-padded to bytes32). `config:get` should show the LZ Labs DVN you
configured and the confirmation count from `layerzero.config.ts`.

Spot-check `dstChainId`:

```bash
RPC_URL="$SEPOLIA_RPC_URL" npx hardhat --network sepolia console
```

```javascript
const b = await ethers.getContractAt("ConfidentialBridge", process.env.SEPOLIA_BRIDGE_ADDRESS);
await b.getDstChainId(40267); // → 80002n
```

Repeat symmetric on Amoy (`getDstChainId(40161)` → `11155111n`).

> The bridge wiring isn't required for the `FHECounter` smoke test in steps
> 5–6 (it's purely local), but you'll want it the moment you move to
> `ConfidentialOFT` or any other cross-chain app.

## Step 4 — Start the mock coprocessor daemon

**Open a dedicated terminal** and keep it running. The daemon is
**live-events-only**: it seeds its cursor at each chain's current head on
startup and never catches up missed events. Any test transaction submitted
before the daemon's `initialised at chain head N` line will be missed.

```bash
npm run mock:daemon
```

Wait until you see two `initialised at chain head …` lines (one per chain):

```
[mock-coprocessor:sepolia]     initialised at chain head 9301245 — only events from block 9301246 onwards will be processed
[mock-coprocessor:polygonAmoy] initialised at chain head 15403200 — only events from block 15403201 onwards will be processed
```

Only proceed once both have printed. Leave the daemon alive for the rest of
the runbook.

## Step 5 — Deploy `FHECounter` on each chain

The `FHECounter` constructor emits a `TrivialEncrypt(0)` event whose handle is
the operand for every subsequent `add()`. That event must be observed by the

```bash
cp addresses/.env.host addresses-sepolia/.env.host
```

### Polygon Amoy

```bash
cp addresses-amoy/{FHEVMHostAddresses.sol,.env.host} addresses/

RPC_URL="$POLYGON_AMOY_RPC_URL" \
  npx hardhat --network polygonAmoy task:deployFHECounter

cp addresses/.env.host addresses-amoy/.env.host
```

Watch the daemon terminal: after each deploy it should log a
`processed blocks … inserted=1` (or more) line on the matching chain within a

## Step 6 — Exercise `FHECounter` and verify the plaintext

`task:runFHECounterDemo` calls `add(amount)` `times` times and prints the

### Sepolia

## Step 8 — Deploy and test `ConfidentialOFT`

`ConfidentialOFT` is the reference cross-chain confidential token that rides
on top of the bridge: each chain holds encrypted per-user balances, and
`send(...)` burns from the sender on the source chain while the destination
chain's `onReceive(...)` mints to the recipient.

Prerequisites for this section:

- Steps 1–3 are done (host stack + bridges deployed and wired on both chains).
- Daemon (step 4) is running — otherwise the daemon won't see the burn/mint
  events and the OFT's cleartext balances won't be observable.

### 8.1 — Deploy on Sepolia

```bash
cp addresses-sepolia/{FHEVMHostAddresses.sol,.env.host} addresses/

RPC_URL="$SEPOLIA_RPC_URL" \
  npx hardhat --network sepolia task:deployConfidentialOFT

cp addresses/.env.host addresses-sepolia/.env.host
```

The new address is appended to `addresses/.env.host` as
`CONFIDENTIAL_OFT_CONTRACT_ADDRESS`. The constructor wires the OFT to the
local bridge it just read from the same file, and sets the deployer wallet as
the OFT's `Ownable2Step` owner.

### 8.2 — Deploy on Polygon Amoy

```bash
cp addresses-amoy/{FHEVMHostAddresses.sol,.env.host} addresses/

RPC_URL="$POLYGON_AMOY_RPC_URL" \
  npx hardhat --network polygonAmoy task:deployConfidentialOFT

cp addresses/.env.host addresses-amoy/.env.host
```

Export the two OFT addresses for the next step:

```bash
export SEPOLIA_OFT_ADDRESS=$(grep '^CONFIDENTIAL_OFT_CONTRACT_ADDRESS=' addresses-sepolia/.env.host | cut -d= -f2)
export POLYGON_AMOY_OFT_ADDRESS=$(grep '^CONFIDENTIAL_OFT_CONTRACT_ADDRESS=' addresses-amoy/.env.host | cut -d= -f2)
```

### 8.3 — Trust each remote OFT as a peer

`ConfidentialOFT.onReceive` rejects any inbound `(srcEid, srcApp)` pair the
owner hasn't whitelisted (`UntrustedPeer`). Call `task:wireConfidentialOFT`
once per direction. The task accepts a regular EVM address and pads it to the
on-chain `bytes32` type internally.

```bash
# Sepolia → trust the Amoy OFT
cp addresses-sepolia/{FHEVMHostAddresses.sol,.env.host} addresses/
RPC_URL="$SEPOLIA_RPC_URL" \
  npx hardhat --network sepolia task:wireConfidentialOFT \
  --remote-eid 40267 \
  --remote-oft "$POLYGON_AMOY_OFT_ADDRESS"

# Amoy → trust the Sepolia OFT
cp addresses-amoy/{FHEVMHostAddresses.sol,.env.host} addresses/
RPC_URL="$POLYGON_AMOY_RPC_URL" \
  npx hardhat --network polygonAmoy task:wireConfidentialOFT \
  --remote-eid 40161 \
  --remote-oft "$SEPOLIA_OFT_ADDRESS"
```

### 8.4 — Verify wiring

```bash
RPC_URL="$SEPOLIA_RPC_URL" npx hardhat --network sepolia console
```

```javascript
const oft = await ethers.getContractAt("ConfidentialOFT", process.env.SEPOLIA_OFT_ADDRESS);
const remoteOftBytes32 = ethers.zeroPadValue(process.env.POLYGON_AMOY_OFT_ADDRESS, 32);
await oft.isTrustedPeer(40267, remoteOftBytes32); // → true
await oft.confidentialBridge(); // → local ConfidentialBridge address
```

Repeat the symmetric check on Amoy (`isTrustedPeer(40161, sepoliaOftAsBytes32)` → `true`).

You can also use the local bridge's `quote(...)` to confirm the LZ pathway
priced correctly, which is a cheap on-chain sanity check that step 3 is
complete:

```javascript
const bridge = await ethers.getContractAt("ConfidentialBridge", process.env.SEPOLIA_BRIDGE_ADDRESS);
// quote(dstEid, srcApp, dstApp, payload, handleList, lzComposeGas, options)
// Use the local OFT as srcApp + a 64-byte payload matching what oft.send wraps
// internally (abi.encode(recipient, actualAmount)) so the fee is realistic.
const dummyPayload = ethers.AbiCoder.defaultAbiCoder().encode(
  ["address", "bytes32"],
  [ethers.ZeroAddress, ethers.ZeroHash]
);
await bridge.quote(40267, process.env.SEPOLIA_OFT_ADDRESS, remoteOftBytes32, dummyPayload, [ethers.ZeroHash], 200000n, "0x");
// → { nativeFee: <bigint>, lzTokenFee: <bigint> }
```

A non-zero `nativeFee` means LZ routes through your configured DVN/Executor
end-to-end. If it reverts with `UnknownDstEid`, step 3.4 wasn't done.

### 8.5 — End-to-end send: seed balance via `mint`, bridge to the other chain

`ConfidentialOFT` exposes an `onlyOwner` `mint(address to, externalEuint64
encryptedAmount, bytes inputProof)` for bootstrapping the initial supply on
each chain (the production token economy is otherwise governed by the
burn-and-mint bridge flow). The encrypted `(handle, inputProof)` bundle is
produced off-chain by the mock coprocessor's `mock:encrypt` CLI, which
emulates the relayer / input-verifier path: it builds a handle byte-identical
to what the production relayer would compute, EIP-712-signs over it with the
configured coprocessor signers, and inserts the cleartext into the mock DB
so the daemon's `VerifyInput` handler resolves it once the tx is mined.

> Prereqs for the encryption step: `CHAIN_ID_GATEWAY`,
> `INPUT_VERIFICATION_ADDRESS`, `NUM_COPROCESSORS`, and
> `PRIVATE_KEY_COPROCESSOR_ACCOUNT_0..N` must be set in the root `.env` (the
> CLI cross-checks each derived address against `COPROCESSOR_SIGNER_ADDRESS_i`
> when both are set, and refuses to build a proof the on-chain
> `InputVerifier` would reject). These same values were used at
> `task:deployInputVerifier` time, so as long as you didn't rotate them
> between the deploy and now, they're already correct.

#### 8.5.1 — Mint an initial balance on Sepolia

```bash
# Resolve a few addresses for convenience.
export DEPLOYER_PRIVATE_KEY=$(grep '^DEPLOYER_PRIVATE_KEY=' .env | sed -E 's/^[^"]*"([^"]*)".*/\1/')
export DEPLOYER_ADDRESS=$(cast wallet address "$DEPLOYER_PRIVATE_KEY")
export SEPOLIA_OFT_ADDRESS=$(grep '^CONFIDENTIAL_OFT_CONTRACT_ADDRESS=' addresses-sepolia/.env.host | cut -d= -f2)
export POLYGON_AMOY_OFT_ADDRESS=$(grep '^CONFIDENTIAL_OFT_CONTRACT_ADDRESS=' addresses-amoy/.env.host | cut -d= -f2)
export SEPOLIA_BRIDGE_ADDRESS=$(grep '^CONFIDENTIAL_BRIDGE_CONTRACT_ADDRESS=' addresses-sepolia/.env.host | cut -d= -f2)

# Build a single euint64=1000 encrypted input bound to (OFT, deployer) on Sepolia
# and insert its cleartext into the mock DB. Prints handle=… inputProof=… on stdout.
pnpm mock:encrypt \
  --contract "$SEPOLIA_OFT_ADDRESS" \
  --user    "$DEPLOYER_ADDRESS" \
  --type    euint64 \
  --value   1000 \
  --host-chain-id 11155111 \
  | grep -E '^(handle|inputProof)=' > enc.out

# Re-export the two outputs for the next step.
set -a; source enc.out; set +a   # exposes $handle and $inputProof
echo "handle:      $handle"
echo "inputProof:  $inputProof (length $(((${#inputProof} - 2) / 2)) bytes — expected 100)"
```

Now call `mint` from a hardhat console. The wallet signing the tx **must** be
the OFT owner (the same `DEPLOYER_PRIVATE_KEY` used to deploy it) AND must be
the same address you passed as `--user` above — both are bound into the
EIP-712 signature.

```bash
cp addresses-sepolia/{FHEVMHostAddresses.sol,.env.host} addresses/
RPC_URL="$SEPOLIA_RPC_URL" npx hardhat --network sepolia console
```

```javascript
const deployer = new ethers.Wallet(process.env.DEPLOYER_PRIVATE_KEY, ethers.provider);
const oft = await ethers.getContractAt("ConfidentialOFT", process.env.SEPOLIA_OFT_ADDRESS, deployer);

const tx = await oft.mint(deployer.address, process.env.handle, process.env.inputProof);
const receipt = await tx.wait();
console.log("mint tx:", tx.hash, "block:", receipt.blockNumber);

// Confirm balance, then verify the plaintext via the daemon.
const balHandle = await oft.balanceOf(deployer.address);
console.log("balance handle:", ethers.toBeHex(balHandle, 32));
```

Wait for the daemon's next `processed blocks …` line, then in a third
terminal:

```bash
pnpm mock:query <balance handle from above>
# → 1000
```

If the result isn't 1000, something went wrong; consult the daemon log
(it should show `VerifyInput`, then a `TrivialEncrypt(0)` from the
uninitialised-balance branch in `FHE.add`, then an `FheAdd` resolving to the
new balance handle).

#### 8.5.2 — Bridge it to Amoy

Once you have a non-zero balance handle on Sepolia:

```javascript
// Continuing in the Sepolia hardhat console:
const remoteOftBytes32 = ethers.zeroPadValue(process.env.POLYGON_AMOY_OFT_ADDRESS, 32);
const bridge = await ethers.getContractAt("ConfidentialBridge", process.env.SEPOLIA_BRIDGE_ADDRESS, deployer);

// Quote the LZ native fee.
// IMPORTANT: oft.send internally calls bridge.send with
//   payload = abi.encode(recipient, actualAmount)
// so the quote MUST use the same payload size or the returned fee will
// underestimate and `_lzSend` will revert. We don't yet know `actualAmount`
// (it's produced inside `_burn`), but any bytes32 works for sizing.
const payload = ethers.AbiCoder.defaultAbiCoder().encode(
  ["address", "bytes32"],
  [deployer.address, balHandle]
);
const fee = await bridge.quote(
  40267,
  process.env.SEPOLIA_OFT_ADDRESS,
  remoteOftBytes32,
  payload,
  [balHandle],
  200000n,
  "0x"
);
console.log("native fee (wei):", fee.nativeFee.toString());

// Bridge the full balance to the deployer on Amoy.
//
// IMPORTANT: LZ V2's `OAppSender._payNative` (both the upgradeable and the
// non-upgradeable variants) requires `msg.value == _nativeFee` EXACTLY — not
// `>=`. Overpay reverts with `NotEnoughNative(uint256 msgValue)` (selector
// 0x9f704120). So always pass the freshly quoted fee verbatim, and keep the
// quote and the send tight (same console session is fine).
const sendTx = await oft.send(
  40267, // Amoy LZ EID
  remoteOftBytes32, // destination OFT (bytes32)
  balHandle, // encrypted amount (the full balance)
  deployer.address, // recipient on Amoy
  200000n, // mintComposeGas
  { value: fee.nativeFee }
);
console.log("send tx:", sendTx.hash);
```

LayerZero delivery on testnet typically takes 30–90s. Track it on
[LayerZero Scan Testnet](https://testnet.layerzeroscan.com/).

When the destination side fires, watch the daemon terminal: it should log a
`HandleBridged` event on Amoy and (after the source-side `FheSub`/`select`
events for `_burn` are processed) a `resolved pending mapping …` line as the
retry queue propagates the cleartext.

Then on Amoy:

```bash
cp addresses-amoy/{FHEVMHostAddresses.sol,.env.host} addresses/
RPC_URL="$POLYGON_AMOY_RPC_URL" npx hardhat --network polygonAmoy console
```

```javascript
const deployer = new ethers.Wallet(process.env.DEPLOYER_PRIVATE_KEY, ethers.provider);
const oft = await ethers.getContractAt("ConfidentialOFT", process.env.POLYGON_AMOY_OFT_ADDRESS, deployer);
const balHandle = await oft.balanceOf(deployer.address);
console.log("Amoy balance handle:", ethers.toBeHex(balHandle, 32));
```

```bash
pnpm mock:query <balance handle on Amoy>
# → 1000
```

A printed `1000` on Amoy confirms: the OFT burned on Sepolia, the bridge
delivered the handle association via LayerZero, the OFT minted on Amoy, and
the mock coprocessor propagated the cleartext end-to-end.

> Bridging a **partial** amount (say 250 of the 1000) works the same way:
> encrypt a separate `euint64=250` bundle on Sepolia and pass that handle to
> `oft.send(...)` instead of the full balance. The on-chain `_burn` will
> decrement Sepolia's balance by 250 and the OFT will mint 250 to the
> recipient on Amoy.

> Want a self-contained alternative that doesn't touch a live testnet? The
> Hardhat fixture suite (`test/bridge/ConfidentialOFT.ts`) mocks both LZ
> endpoints in-process and exercises the full round-trip:
>
> ```bash
> npx hardhat test test/bridge/ConfidentialOFT.ts
> ```

## Useful one-liners

| Need                                                       | Command                                                                                                                   |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Stop the daemon cleanly                                    | `Ctrl-C` in its terminal                                                                                                  |
| Wipe accumulated handles (e.g. `FheRand` poisoning the DB) | `pnpm mock:reset` (do it while the daemon is stopped)                                                                     |
| Re-query any handle later                                  | `pnpm mock:query 0x…`                                                                                                     |
| Build an encrypted input for an `FHE.fromExternal` call    | `pnpm mock:encrypt --contract <addr> --user <addr> --type euint64 --value 1000 --host-chain-id 11155111`                  |
| Inspect current LZ peers                                   | `cd lz-wiring && pnpm peers:get`                                                                                          |
| Inspect current LZ config (libraries/DVN/Executor)         | `cd lz-wiring && pnpm config:get`                                                                                         |
| Re-apply LZ wiring after `layerzero.config.ts` edits       | `cd lz-wiring && pnpm wire:dry` then `pnpm wire`                                                                          |
| Re-upgrade only the bridge implementation later            | `LZ_ENDPOINT_ADDRESS=… RPC_URL=… npx hardhat --network <name> task:deployBridge`                                          |
| Re-deploy `ConfidentialOFT` only                           | `RPC_URL=… npx hardhat --network <name> task:deployConfidentialOFT`                                                       |

| -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Daemon logs `Operand handle not found …` on the first `add()` call                                 | Counter was deployed **before** the daemon initialised at head                                                                                                                                                                             | Stop daemon → `pnpm mock:reset` → start daemon → wait for the `initialised at chain head` lines → re-deploy `FHECounter` → re-run demo                                                                                                |

| `task:wireBridge` reverts `OwnableUnauthorizedAccount`                                             | Wallet is not the bridge's operational owner                                                                                                                                                                                               | Use the same `DEPLOYER_PRIVATE_KEY` that ran `task:deployBridge`, or `transferOwnership` first                                                                                                                                        |
| `pnpm wire` fails: `Could not find a deployment for address …`                                     | `prewire` step couldn't synthesise the hardhat-deploy artifact                                                                                                                                                                             | Re-export `SEPOLIA_BRIDGE_ADDRESS` / `POLYGON_AMOY_BRIDGE_ADDRESS` in the **same shell** that runs `pnpm wire`                                                                                                                        |

| `oft.mint(...)` reverts with `InvalidSigner` / `SignatureThresholdNotReached` from `InputVerifier` | `mock:encrypt` signed with keys not registered on-chain in `InputVerifier`                                                                                                                                                                 | Check `PRIVATE_KEY_COPROCESSOR_ACCOUNT_*` derives the same addresses as the on-chain `inputVerifier.getCoprocessorSigners()`. | `oft.mint(...)` reverts with `InvalidContractAddress` / `InvalidUserAddress`                       | `--contract` / `--user` passed to `mock:encrypt` didn't match the OFT / caller                                                                                                                                                             | Re-run `mock:encrypt` with `--contract <OFT>` and `--user <tx-sender>` (= `DEPLOYER_ADDRESS` when calling `mint`)                                                                                                                     |
| Daemon errors: `Operand handle not found in mock DB: <handle>` right after a `mint`                | `mock:encrypt` was run with `--host-chain-id` not matching the network you broadcast on (bytes 22..29 of the handle then don't match `block.chainid` and the daemon stores the cleartext under a different key than `VerifyInput` queries) | Re-run `mock:encrypt` with `--host-chain-id 11155111` on Sepolia / `80002` on Amoy, then retry `mint`                                                                                                                                 |

## Architecture quick reference

- **Two ownership layers on the bridge** (see `contracts/bridge/ConfidentialBridge.sol`):
  - _Operational owner_ (`onlyOwner` via `OAppCoreUpgradeable`): `setPeer`,
    `setDstChainId`, `grantFallbackPlaintext`, LZ delegate. Set to
    `deployer.address` by `task:deployBridge`.
  - _Upgrade owner_ (`onlyACLOwner` via `ACLOwnable`): authorizes UUPS
    upgrades. Sourced from `ACL.owner()`.
- **Bridge address baked into the ACL.** `confidentialBridgeAdd` lives in
  `addresses/FHEVMHostAddresses.sol` and is written by
  `task:deployEmptyUUPSProxies` before the ACL is compiled — so the ACL
  implementation has the right constant from the very first deploy, no
  follow-up upgrade needed. Each chain has its own ACL implementation, so
  bridge addresses can differ across chains without issue.
- **Mock coprocessor is live-events-only.** No history catch-up, no
  persisted cursor. Restarting the daemon resets to whatever block is head
  at that moment. The `ciphertexts` table persists across restarts so
  `pnpm mock:query` can read from a separate process.
